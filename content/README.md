# Content Notes

The markdown files in this folder are documentation inputs for humans and AI tools.

They do not need frontmatter.

They do not need section codes, component IDs, or strict logical mapping.

Write them as plain markdown that clearly describes:

- what the page is
- the important copy
- any useful metadata such as title, description, category, date, or image
- the intended structure of the page when that context matters

`home.md` is kept as a single file on purpose so the full homepage story can be read in one pass.

Blog posts stay one file per article, plus `blog/index.md` for the blog landing page.
